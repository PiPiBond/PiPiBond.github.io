                            # 生命周期

                            • onCreate: 创建活动。把页面布局加载进内存，进入了初始状态。
                            • onStart: 开始活动。把活动页面显示在屏幕上，进入了就绪状态。
                            • onResume: 恢复活动。活动页面进入活跃状态，能够与用户正常交互，例如允许响应用户的点击动作、允许用户输入文字等等。
                            • onPause: 暂停活动。页面进入暂停状态，无法与用户正常交互。
                            • onStop:停止活动。页面将不在屏幕上显示。
                            • onDestroy:销毁活动。回收活动占用的系统资源，把页面从内存中清除。
                            • onRestart: 重启活动。重新加载内存中的页面数据。

                            # 启动模式

                            每一个 Activity 都可以在 AndroidManifest.xml 设置一个独立的启动模式。

                            # Standard（默认）

                            标准模式，无论何时启动哪个活动，都是重新创建该页面的实例并放入栈顶。如果不指定 launchMode 属性，则默认为标准模式

                            # SingleTop（栈顶复用）

                            启动新活动时，判断如果栈顶正好就是该活动的实例，则重用该实例；否则创建新的实例并放入栈顶，也就是按照 standard 模式处理

                            # SingleTask（栈内复用）

                            启动新活动时，判断如果栈中存在该活动的实例，则重用该实例并清除位于该实例上面的所有实例；否则按照 standard 模式处理

                            # SingleInstance（全局唯一）

                            启动新活动时，将该活动的实例放入一个新栈中，原栈的实例列表保持不变

                            # 启动标识

                            常用 FLAG

                            FLAG 标识说明
                            FLAG_ACTIVITY_CLEAR_TOP这个 FLAG 就相当于加载模式中的 SingleTask，如果启动的 Activity 存在当前的栈中，系统会把要启动的 Activity 之上的 Activity 全部弹出栈空间，然后把 Intent 作为一个新的 Intent 传给这个 Activity
                            FLAG_ACTIVITY_NEW_TASK开辟一个新的任务栈，标志启动在一个新的 Task 中。当与 FLAG_ACTIVITY_CLEAR_TOP 一起用，与启动模式"singleTask"一致
                            FLAG_ACTIVITY_SINGLE_TOP这个 FLAG 就相当于加载模式中的 singleTop
                            FLAG_ACTIVITY_NO_HISTORY栈中不保存新启动的活动实例，例如 A 启动 B，B 启动 C，C finish ()后看到的界面是 A，这个时候 B 也同时被 finish()了
                            FLAG_ACTIVITY_CLEAR_TASK跳转到新页面时，栈中的原有实例都被清空，这个标志只能与 FLAG_ACTIVITY_NEW_TASK 一起使用。

                            # 启动

                            启动当前应用的程序

                            Intent intent = new Intent(serviceName.this, serviceName.class);

                            通过包名启动其它程序

                            String packageName = "com.example.package";
String serviceName = "com.example.package.XXXXXXX";
Intent intent = new Intent();
intent.setComponent(new ComponentName(packageName, serviceName));
startService(intent);

                            通过 setAction 启动其它程序

                            Intent intent = new Intent();
intent.setAction("com.example.action.MY_ACTION");
// 指定应用
// intent.setPackage("com.example.package"); 
startActivity(intent);

                            发送短信（调用系统短信应用）

                            Intent intent = new Intent(Intent.ACTION_SENDTO);
intent.setData(Uri.parse("smsto:" + "10086"));
intent.putExtra("sms_body", "Hello");
startActivity(intent);

protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_second);

    resp = findViewById(R.id.resp);

    // Code for button listener...

    register = registerForActivityResult(new ActivityResultContracts.StartActivityForResult(), result -> {
        if (result.getResultCode() == RESULT_OK) {
            Intent intent = result.getData();
            String responseData = intent.getStringExtra("responseData");
            resp.setText(responseData);
        }
    });
}

        .append("isMale: ").append(isMale).append("\n")
        .append("score: ").append(score).append("\n")
        .append("hobbies: ");
for (String hobby : hobbies) {
    stringBuilder.append(hobby).append(",");
}
data.setText("received data:"+stringBuilder.toString());
